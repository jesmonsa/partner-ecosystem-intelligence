# 🧠 Skills — Expertos por dominio

Rutas para formar especialistas profundos que actúan como multiplicadores dentro de cada partner, empaquetadas como **skills reutilizables** que la IA y los equipos consumen bajo demanda. Conectan con el Pilar 1.

> Una *skill* = paquete con (a) ruta de certificación, (b) prompts y plantillas, (c) arquitecturas de referencia, (d) checklist de "experto listo".

| Skill | Dominio | Certificación base | Activos vinculados |
|-------|---------|--------------------|--------------------|
| **DB & Data** | Autonomous, Exadata, MySQL HeatWave, GoldenGate | OCI Data Management Associate | Prompts BOM/migración + arquitecturas de datos |
| **Cloud Native** | OKE, serverless, microservicios | OCI Foundations → Architect Associate | Arquitecturas OKE/serverless one-click |
| **IA / GenAI** | OCI Generative AI, Vector Search, agentes | OCI AI Foundations | AI Harness + prompts de radiografía |
| **Seguridad** | IAM, Cloud Guard, Zero Trust, hardening | OCI Security Associate | Checklist de seguridad de IaC |
| **Migración** | OCM, ZDM, DMS, HCX, 7 Rs | OCI Architect Associate | Toolkit + readiness assessment |
| **Observabilidad / DevOps** | Monitoring, Logging, APM, Terraform, GitOps | OCI DevOps Professional | Arquitecturas DevOps + runbooks |

## Cómo opera una skill

1. **Cohorte** por país/VAD con meta de certificación trimestral.
2. **Ruta**: Foundations → Associate → Specialty por rol.
3. **Práctica**: despliegue real desde el catálogo `oracle-cloud-latam`.
4. **Evidencia**: seguimiento de certificaciones por partner.
5. **Multiplicación**: el experto certificado habilita a su equipo.

## Roadmap de skills

- **Mes 1:** DB & Data y Cloud Native.
- **Mes 2:** Migración e IA/GenAI con pilotos.
- **Mes 3:** Seguridad y Observabilidad/DevOps + badges regionales.
